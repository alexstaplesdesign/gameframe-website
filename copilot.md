# Copilot Instructions for This Project

This is a production website built with:
- Static HTML files
- Tailwind CSS for all styling
- No JavaScript frameworks
- No templating engines

General Rules:
- Do not introduce React, Vue, JSX, or any framework concepts.
- Do not rewrite entire files unless explicitly asked.
- Make small, targeted, intentional changes.
- Preserve existing structure and class ordering when possible.
- Do not add inline styles or custom CSS unless requested.

Tailwind CSS Usage:
- Use Tailwind utility classes only.
- Prefer standard spacing and sizing utilities.
- Avoid arbitrary values unless absolutely necessary.
- Keep class lists readable and purposeful.
- Do not over-engineer simple layouts.

HTML Structure:
- Use semantic HTML elements where appropriate.
- Avoid unnecessary wrapper divs.
- Keep markup clean and minimal.
- Maintain accessibility basics (alt text, labels, button types).

Design & Layout:
- Think like a designer, not just a coder.
- Pay attention to spacing, hierarchy, alignment, and balance.
- Ensure responsive behavior by default.
- Favor clean, modern, professional presentation.

Workflow:
- Briefly explain intended changes before writing code.
- If something is unclear, ask at most one clarifying question.
- Treat existing pages as production-ready.
- Prefer incremental refinement over refactoring.
